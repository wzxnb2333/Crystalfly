# Crystalfly 0.3.0

## 中文

- 扫描 Modding API、旧版平铺目录和 BepInEx 插件目录，并显示没有收据的外部 Mod。
- 支持外部 Mod 接管、收据 v2 迁移、文件哈希完整性检查、精确版本修复、本地重新导入与接受当前文件。
- 支持固定 Mod；批量卸载和无用前置建议会跳过固定项。
- 启动页持续显示完整性问题。Mod 文件与依赖问题可在确认后强制启动，其他启动阻止项保持不可绕过。
- “不再提醒”绑定实例、Mod、问题类型、文件路径和当前哈希；问题变化后自动失效。
- 新增全局离线模式，目录与下载只使用已验证缓存，下载队列在恢复在线后继续。

## English

- Discovers external Mods across modern Modding API, legacy flat layouts, and BepInEx plugin directories.
- Adds receipt schema v2 migration, explicit takeover, integrity reports, exact-version repair, local re-import, and accept-current workflows.
- Adds Mod pinning and suggestion-only unused dependency cleanup.
- Keeps launch issues visible and permits force launch only for Mod file or dependency problems.
- Warning acknowledgements are invalidated when the instance, Mod, issue, path, or current file hash changes.
- Adds global offline mode with verified-cache-only reads and resumable waiting download groups.
