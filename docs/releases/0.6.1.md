# Crystalfly 0.6.1

## 实例配置与存档

- 恢复当前实例隔离的 `AppConfig.ini` 编辑，保留未知 INI 字段并使用原子写入。
- 恢复当前实例和命名快照的存档编辑，只展示和修改 `user1.dat` 至 `user4.dat`。
- 存档解密、JSON 展开和列表构建改为异步执行，进入编辑器时主窗口保持响应。
- 空存档实例会显示明确空状态；切换实例或启动游戏会清除旧编辑上下文。
- 更新快照存档后同步刷新快照哈希，避免元数据与实际内容不一致。

## 界面与加载反馈

- 配置编辑入口恢复到实例详情导航，存档编辑入口恢复到存档快照页面。
- 全局和存档加载动画统一为居中的放大骑士行走动画。
- 加载器隐藏、窗口隐藏或系统启用减少动态效果时暂停动画。
- 大型存档字段列表启用虚拟化，降低渲染开销。
- 修复关闭窗口后单实例命名管道释放捕获已停止的界面同步上下文，导致无窗口后台进程残留的问题。

## 目录与依赖防护

- 远程目录在写入缓存前过滤客户端未识别的构建、Loader、Mod 和速通条目。
- 未知构建的占位哈希不会污染有效目录缓存；非法哈希、URL 或依赖仍触发完整回退。
- Mod 删除与依赖修复对话框恢复父子树、连接线和节点状态图标。

所有配置和存档操作都限定在所选实例或其命名快照范围内，不会读取或写入其他实例的数据。
