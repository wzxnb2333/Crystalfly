# Crystalfly 0.6.1 补齐设计

## 目标

将主开发工作树中尚未提交的产品功能安全集成到最新 `main`，发布为 `0.6.1`，同时保留 0.6.0 已有的协议、更新器、预设、Mod 管理和下载队列能力。

## 已确认遗漏范围

- 主窗口布局、主题、对齐和窗口交互。
- 游戏配置编辑、INI 解析与隔离写入。
- 存档编辑及实例 LocalLow 范围限制。
- 小骑士加载动画与资源定位、缩放。
- 依赖树、快照、目录缓存和对应测试。

## 集成方式

以远程 `main` 的 `758058a` 为基线，在 `fix/complete-ui-config-animation` 分支中进行逐文件三方整合。来源工作树为 `E:\AI_collection\crystalfly`，该目录保持原样；只提取运行时代码、测试和被代码引用的资源。

`.qoder`、临时分析报告、无代码引用的试验内容不会进入产品提交。对 `MainWindow`、主题、主 ViewModel、构建脚本等高冲突文件，先比较 `7cc9be4` 到来源工作树的改动，再人工合并到最新基线，避免回退已经发布的功能。

## 用户体验约束

- 保持 Semi/Ursa 与 PCL2 紧凑信息架构。
- 加载动画固定在内容区视觉中心，使用已有骑士资源并提高可见尺寸。
- 配置和存档编辑仅操作当前实例的隔离数据；存档编辑只显示该实例的四个存档槽。
- 既有中英文、亮暗主题、窗口控制、实例与 Mod 工作流保持可用。

## 验收与发布

- 为每项缺失行为先补充或迁移回归测试，再移植实现。
- 运行完整 Release 构建、全部测试、发布脚本、安装脚本和 `git diff --check`。
- 通过实际安装目录验证启动、动画、配置编辑、四存档隔离与既有实例数据保留。
- 创建新的 `v0.6.1` Release；保持 `v0.6.0` 的发布资产和哈希历史不变。
